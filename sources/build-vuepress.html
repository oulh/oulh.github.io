# 快速本地部署

前提条件

VuePress 需要 Node.js >= 8.6

  1. 创建并进入一个新目录
mkdir vuepress-starter && cd vuepress-starter
  1. 使用你喜欢的包管理器进行初始化
yarn init # npm init
  1. 将 VuePress 安装为本地依赖
yarn add -D vuepress # npm install -D vuepress
  1. 创建你的第一篇文档
mkdir docs && echo '# Hello VuePress' > docs/README.md
  1. 在 package.json 中添加一些 scripts
{
  "scripts": {
    "dev": "vuepress dev docs",
    "build": "vuepress build docs"
  }
}
  • vuepress dev docs ：开启本地服务
  • vuepress build docs：构建静态文件

执行yarn dev或npm run dev 时，自动执行 “dev” 中的命令，即 vuepress dev docs

  1. 在本地启动服务器
yarn dev # npm run dev

访问运行结果所示的地址和端口，即可打开站点首页，但这时页面什么都没有，因为标题、导航栏、侧边栏等配置都要自己手动添加。下面就进行主题的配置吧！

# 主题配置

​ 在此前，先看一下官方文档指南中的目录结构吧。vuepress的主题配置主要在docs/README.md和docs/.vuepress/config.js 文件中。但这时docs目录下几乎啥都没有，别担心，以后会有的。现在就在docs下创建.vuepress/config.js吧！

（执行命令npm run build或yarn build也会生成.vuepress目录）

# 一、默认主题配置

参照官方文档默认主题配置，配置一下首页试试吧！

在根级 README.md，即docs/README.md中写入以下内容：

点击查看

home: true
heroImage: /hero.png	#即.vuepress/public/hero.png
heroText: Hero 标题
tagline: Hero 副标题
actionText: 快速上手 →
actionLink: /zh/guide/
features:
- title: 简洁至上
  details: 以 Markdown 为中心的项目结构，以最少的配置帮助你专注于写作。

- title: 高性能
  details: VuePress 为每个页面预渲染生成静态的 HTML，同时在页面被加载的时候，将作为 SPA 运行。

在终端里Ctrl+C关闭服务，再用之前的命令开启服务，刷新网页。

# 二、主题安装和使用

这里就用当前github上找到的vuepress最热的主题：vuepress-theme-reco ，vuepress-theme-reco-doc

安装和使用方法在文档中说得很清楚了；添加或修改完配置，要重启本地服务，再刷新网页。

上一份简单配置来参考一下吧

docs/README.md

---
home: true
bgImage: /bg.svg #首页背景图片
bgImageStyle: {
    
}
heroImage: /logo.png  #标题logo
heroText: null  #标题
tagline: 一将成，万骨枯
---

.vuepress/config.js

点击查看 
module.exports = {
  theme: 'reco',
  themeConfig:{
	  type: 'blog', //reco主题的首页blog模式
	  authorAvatar: '/cc.jpg', //侧边栏作者头像
	  logo: '/logo.png'	,//导航栏logo
	//导航栏设置
	  nav: [
			{ text: '主页', link: '/'},
			{
				text: '联系',
				ariaLabel: 'Contact Menu',
				items: [
				{ text: '思否', link: 'url' },
				{ text: 'github', link: 'url' }
				]
			}
	 ],
	//博客配置
	  blogConfig: {
		category: {
			location: 2,     // 在导航栏菜单中所占的位置，默认2
			text: '分类' // 默认文案 “分类”
			},
		tag: {
			location: 3,     // 在导航栏菜单中所占的位置，默认3
			text: '标签'      // 默认文案 “标签”
			}
		},

  }
}

# 发布文章

文章以md文件的形式存放在项目下，经过构建命令渲染成html文件，保存在.vuepress/dist目录下，dist目录就是站点的根目录。

可以在docs目录下创建一个文件夹，专门存放md文件。构建过程中将生成同样的目录结构，所以如果改变md文件的位置，也会改变文章访问链接。

  1. 新建文章，如build-vuepress.md

以下myblog是我的项目根目录：

md文件的顶部必须要有front master，即文章标题、日期等设置。

---
title: Blogging Like a Hacker
lang: en-US
---

  1. 执行构建命令

    yarn build 或 npm run build

  2. 重启本地服务

yarn dev 或 npm run dev
  1. 访问上面命令返回的地址，首页文章列表有新文章更新则说明成功了。

访问本地服务的网页时，对文章的修改可以实时刷新

到此vuepress本地部署就完成了😀

# vuepress远程部署git

# 一、部署到github或gitee

官方文档 ：部署

主要方式是写个脚本，build完后，用git把dist目录的全部文件push到仓库。以后每次执行deploy.sh脚本，就可以实现构建并部署。

点击查看 
#!/usr/bin/env sh

# 确保脚本抛出遇到的错误
set -e

# 生成静态文件
npm run build

# 进入生成的文件夹
cd docs/.vuepress/dist

# 如果是发布到自定义域名
# echo 'www.example.com' > CNAME

git init
git add -A
git commit -m 'deploy'

#发布到gitee
git push -f git@gitee.com:oolh/oolh.git master

#发布到github
git push -f git@github.com:oulh/oulh.github.io.git master


# 如果发布到 https://<USERNAME>.github.io
#git push -f git@github.com:<USERNAME>/<USERNAME>.github.io.git master

# 如果发布到 https://<USERNAME>.github.io/<REPO>
# git push -f git@github.com:<USERNAME>/<REPO>.git master:gh-pages

cd -

# 二、部署到腾讯云COS

可以把项目push到腾讯的coding中，再在coding上创建持续集成计划，每次push都自动把dist目录下的文件上传到腾讯云COS。这样一来，可以同时把网站文件备份在coding上。这个方法经实践过是可行的。

更多的配置可以参考 vuepress中文文档 👈
